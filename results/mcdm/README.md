## How to Regenerate

### Step 1: Run MCDM Evaluation

```bash
python src/mcdm_evaluation.py
```

**Outputs:**
- `supplier_scores_for_rl.npy` — Binary array for RL
- `dimensional_scores.csv` — Human-readable scores per dimension
- `fbwm_weights.csv` — Fuzzy BWM weights (16 criteria)
- `ftopsis_rankings.csv` — Complete FTOPSIS rankings

### Step 2: Verify Output

```bash
python -c "import numpy as np; print(np.load('results/mcdm/supplier_scores_for_rl.npy'))"
```

**Expected Output:**
```
[0.15231098 0.62373179 0.79048104 0.51976268 0.78374438]
```

### Step 3: Run RL Experiments

```bash
python src/run_experiment.py
```

**What Happens:**
- Automatically loads `supplier_scores_for_rl.npy`
- Updates all scenario configurations (Stable, Volatile, Shock)
- Trains M2/M3 models with FTOPSIS-informed state space
- Evaluates sustainability-aware policies

## File Descriptions

### 1. `supplier_scores_for_rl.npy` (Critical)

**Format:** NumPy binary array (float64)
**Shape:** (5,)
**Purpose:** Provides FTOPSIS sustainability scores for RL state space (obs[11:16])

### 2. `dimensional_scores.csv`

**Format:** CSV with 5 rows x 5 columns
**Purpose:** FTOPSIS scores aggregated by dimension (0–10 scale)

### 3. `fbwm_weights.csv`

**Format:** CSV with 16 rows (one per criterion)
**Purpose:** Fuzzy Best-Worst Method weights with consistency validation
**CR:** 0.0442 (well below the 0.10 threshold)

### 4. `ftopsis_rankings.csv`

**Format:** CSV with 5 rows (one per supplier)
**Purpose:** Full FTOPSIS analysis with ideal/anti-ideal distances

### 5. `supplier_raw_scores_table.csv`

**Format:** CSV with 5 rows x 16 columns
**Purpose:** Raw supplier performance scores before FTOPSIS normalisation

## Integration with RL Pipeline

```
mcdm_evaluation.py (Phase I)
       |
  Calculates FTOPSIS scores
       |
  Saves to supplier_scores_for_rl.npy
       |
run_experiment.py (Phase II)
       |
  Loads supplier_scores_for_rl.npy
       |
  Updates BASE_CONFIG["ftopsis_scores"]
       |
  Creates SupplyChainEnv with scores
       |
  State space includes FTOPSIS (obs[11:16])
       |
  M3 learns sustainability-aware policies
```

---

## Important Notes

### 1. File Dependency

`run_experiment.py` **requires** `supplier_scores_for_rl.npy` to exist.

If the file is missing, run `python src/mcdm_evaluation.py` first.

### 2. Data Consistency

All files in this directory are generated from the **same MCDM run**.
If you update `mcdm_evaluation.py` (e.g., change criteria weights), you must regenerate all files.

### 3. Version Control

These output files are tracked in Git for reproducibility. The source of truth is `mcdm_evaluation.py`.
